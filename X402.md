# x402 pay-per-use for IoT actions

[x402](https://docs.cdp.coinbase.com/x402/welcome) is an open standard from Coinbase that reuses HTTP's `402 Payment Required` status to embed on-chain micropayments directly in an API call. DOBI uses it so device owners can charge a small fee per action invocation on Base.

## Why this matters for IoT

Most IoT monetization stories require a billing database, a customer portal, and a payment processor. With x402:

- **No Stripe, no subscriptions.** A caller's wallet pays directly to the device owner's wallet on Base.
- **Metering is the HTTP response.** If the caller hasn't paid, the server returns 402 with the payment details. Pay, then retry.
- **Non-custodial.** DOBI never holds funds. The caller signs the transfer; the backend verifies it on-chain; the device action runs.

Typical rates people set: `$0.001 – $0.05 USDC-equivalent DOBI per call` — low enough not to matter for a human user, high enough that a script abusing an API pays real money.

## End-to-end flow

1. **Device owner enables x402** on a device by toggling `x402_enabled: true` and setting `x402_price` + `x402_wallet` (or inherits the owner wallet). Do this in the wizard when creating a DAM, or in the x402 panel on a device's detail page.
2. **Customer hits the gated action** with `POST /api/x402/:asset_id/pay?token=<session>`.
3. **Backend replies with HTTP 402**:
   ```json
   {
     "error": "Payment Required",
     "payment_id": "…uuid…",
     "asset_id": "charger-01",
     "payment_details": {
       "wallet_address": "0x…",
       "amount": 30000,
       "token": "DOBI",
       "token_address": "0x…",
       "chain_id": 8453,
       "chain_name": "Base"
     }
   }
   ```
4. **Customer's wallet transfers the tokens on Base.** Any wallet that supports ERC-20 transfers works; DOBI emits an `ethereum:<token>@<chain>/transfer?address=<to>&uint256=<amount>` deep link so mobile wallets one-tap through it.
5. **Customer confirms** via `POST /api/x402/confirm` with the tx hash. Backend verifies the on-chain transfer, then executes the action, then returns the action result in the same response.

## Trying it in the dashboard

On any device with `x402_enabled: true`, the Actions tab shows a **Try pay-per-use** panel. Click it → a 402 is returned → the `X402PaymentModal` opens with the real payment details → copy the recipient / open the wallet link / paste your tx hash → action runs.

## Protocol gotchas

- **Session expiry.** Each 402 challenge issues a session that expires in ~5 min. After that, a new 402 is required.
- **Replay protection.** Each payment_id can only settle one action. DOBI uses JSONB containment on the actions table, so no LIKE-injection tricks.
- **Chain verification.** The backend re-queries Base for the tx receipt; spoofing a tx hash doesn't get you the action.
- **Refunds.** There isn't one. Don't price an action higher than it's worth.

## When to turn this on

- ✅ You're exposing a public endpoint and want to prevent free-rider scraping.
- ✅ You're selling access to a physical resource (charger session, data feed, actuator).
- ❌ You and a few teammates are polling the same device — just use auth.
